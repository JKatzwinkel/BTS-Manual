HB-Workflow: 

- Account erstellen auf Github. 
- Github Desktop installieren und anmelden.
- Repository herunterladen, einen lokalen Branch klonen (ex. gunner_1)
- DITA-Datein in Oxygen aktualiesieren.
- Jeden Tag einen "commit" auf lokalen Branch ("gunner_1"). 
- Jede Woche einen "pull request" zu dem remote "master".  
- Jede Woche werden "pull requests" mit remote Master gemerged.  
- Max aktualisiert lokalen Master-branch, klont davon einen eigenen Branch und erzeugt daraus ein neues PDF / HTML Projekt. 
- Max packt die neuen HTML Seiten auf den Server.
