Creating / modifying a database object

The following functions are available to the user with administrator privileges over a given corpus.

Creating a database object

  1. Choose the corpus from the corpora tree where you wish your object to appear. You can add an object to a corpus or to another already existing object.
  2. Click "Add Text Corpus Object" in the toolbar. This action will add an object as a subentry of the chosen corpus. NOTE: currently the new objects and texts will be saved in your main working corpus and not in the selected corpus if they differ. The new objects will appear in the selected corpus though.
  3. Name your object in the Main Tab of the Passport Editor. (The "Name" input field will appear red until it is named.) You can actually start working without naming your object and name it later. Until it is named, it will appear in the corpora tree as "BTST Object"- its default name.
  4. Define the Type and in case also the Subtype of the object. You can and you also should give as much informations as you can by using the tabs of the Passport Editor.

Creating a text

  1. Select a location in the Corpus Navigator (on the left of the default User Interface) where you'd like to allocate your text. You can add a text to an object or even to an already existing text.
  2. Then click "Add Text" in the Corpus Navigator toolbar. Depending on the object you selected, a "child"-text will be added in the next lower hierarchical status in relation to it. In the Navigator it will be named: "BTS Text" (in order to see it, you might have to open the parent object).
  3. Name your text in the Main Tab of the Passport Editor. (The "Name" input field will appear red until it is named.) You can actually start working without naming your text and name it later. Before it is named, it will appear in the corpora tree as "BTS Text"- its default name.

Moving database objects

To change a physical location of a database object (marked in square brackets beside the name) from one project or corpus to another, right click the chosen item and select "Move among projects" in the appearing context menu. In the opened window the current physical location is displayed. Alter the entries as you wish (upper field for a new project and lower field for a new corpus in the project). The change of physical location does not affect the position of the database object in the Navigator tree.

Deleting database objects

To delete a database object (corpus, object, text) select it and then click on "Delete". It will be moved to the bin ("Bin" tab in the Corpus Navigator window), where you have the opportunity to restore it (right click the object, context menu, "Restore") or to delete it permanently. All subordinate elements will be deleted as well. Upon closing or restarting the BTS all the item in the Bin will be permanently deleted.
