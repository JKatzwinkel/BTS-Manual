Grammar check

The BTS is capable of checking a transliteration for errors by using system grammar rules. This does not mean that it can recognize false lemmata or transliteration errors and correct them. In fact, it follows the internal logic according to which specific signs can only occur in specific combinations. It detects disagreements with this logic and marks them:

Images/Ansicht_mit_Fehlern.jpg

Errors are marked by a dark red encircled rectangle on the right next to the scroll bar of the Transliteration tab. There are also light red encircled rectangles. These show the location of a rubrum (in this example l.19 and l.37). A red wavy line in the transliteration field marks errors in the Transliteration Tab window itself. On mouse-over at the red square on the left of the Text Editor you can see a ToolTip with information about the cause of the error.

Images/Ansicht_mit_Fehlern_on_mouse_over.jpg

In this case a half-bracket was opened, but the second one is missing. If you insert the required symbol the error message will disappear:
